ghost를 이용하여 사이트 개편

05 June 2015

2년동안 사용하던 saltfactory's blog를 Jekyll을 이용하여 GitHub Pages로 이전하였다.

블로그 이전에 관한글은 Tistory에서 Jekyll을 이용하여 GitHub Pages로 블로그 이전 글을 참조하면 된다.

Jekyll과 GitHub Pages로 만든 블로그 운영은 내가 기대했던 것보다 장점도 많고 단점도 많은 것 같다. 그래서 메인 사이트는 ghost를 사용해보기로 했다. 블로그 플랫폼 중에 최근 가장 인기 있는 두가지 플랫폼을 사용해보고 나중에 하나로 합치거나, 꼭 그렇지 않아도 이 두가지의 장,담점을 경험해보기 위해서 이다. 일단 두 플랫폼모두 내가 추구하는 철학과 어울리는 환경을 제공한다.

ghost와 Jekyll의 장점은 다음과 같다.

  • 빠르다
  • Pretty하고 Simple하다
  • 글을 작성하는 것이 재밌다.
  • Theme, Plugins 를 지원한다
  • 커뮤니티가 활발하다

ghost와 Jekyll의 차이점을 간단히 살펴보면 다음과 같다.

Programming Language

  • ghost : Node.js
  • Jekyll : Ruby

Database

  • ghost : SQLite, MySQL, PostgreSQL을 지원한다. default는 SQLite로 설정이 된다.
  • Jeyll : 데이터베이스가 없음, .md 파일 자체를 데이터베이스처럼 사용한다.

이러한 구조 때문에 ghost는 웹에서 Markdown을 작성할 수 있는 editor를 제공하고 있고, Jekyll은 특별한 에디터없이 static 파일을 _post라는 디렉토리에 생성하면 바로 웹 페이지가 만들어지도록 한다.
