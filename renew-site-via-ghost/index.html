ghost를 이용하여 사이트 개편

05 June 2015 on ghost

2년동안 사용하던 saltfactory's blog를 Jekyll을 이용하여 GitHub Pages로 이전하였다.

블로그 이전에 관한글은 Tistory에서 Jekyll을 이용하여 GitHub Pages로 블로그 이전 글을 참조하면 된다.

Jekyll과 GitHub Pages로 만든 블로그 운영은 내가 기대했던 것보다 장점도 많고 단점도 많은 것 같다. 그래서 메인 사이트는 ghost를 사용해보기로 했다. 블로그 플랫폼 중에 최근 가장 인기 있는 두가지 플랫폼을 사용해보고 나중에 하나로 합치거나, 꼭 그렇지 않아도 이 두가지의 장,담점을 경험해보기 위해서 이다. 일단 두 플랫폼모두 내가 추구하는 철학과 어울리는 환경을 제공한다.

ghost와 Jekyll의 장점은 다음과 같다.

  • 빠르다
  • Pretty하고 Simple하다
  • 글을 작성하는 것이 재밌다.
  • Theme, Plugins 를 지원한다
  • 커뮤니티가 활발하다

ghost와 Jekyll의 차이점을 간단히 살펴보면 다음과 같다.

Programming Language

  • ghost : Node.js
  • Jekyll : Ruby

두가지 모두 매력적인 프로그래밍 언어이고 github에서도 가장 인기 있는 언어이다. 이 두 언어는 모두 NPM과 RubyGem을 사용하여 빠르게 프로젝트를 진행할 수 있는 유용한 라이브러리들을 설치할 수 있다.

Database

  • ghost : SQLite, MySQL, PostgreSQL을 지원한다. default는 SQLite로 설정이 된다.
  • Jeyll : 데이터베이스가 없음, .md 파일 자체를 데이터베이스처럼 사용한다.

이러한 구조 때문에 ghost는 웹에서 Markdown을 작성할 수 있는 editor를 제공하고 있고, Jekyll은 특별한 에디터없이 static 파일을 _post라는 디렉토리에 생성하면 바로 웹 페이지가 만들어지도록 한다.

다른 서비스간의 데이터를 이전하기 위해 Markdown 형식이 더 효율적이다.

이번에 블로그를 이전하면서 절실하게 깨닫게 된 것이 바로 HTML으로 저장된 컨텐츠를 다른 서비스로 이전하기는 어렵다는 것이다. Jekyll은 Markdown 파일 자체를 사용하기 때문에 이후에 이전을 하더라도 이 파일들을 그대로 사용하거나 HTML으로 변환하여 사용하면 된다. ghost도 export를 하면 json 파일로 export를 하게 되는데 markdown과 html 프로퍼티 두가지 모두 가져올 수 있다. ghost의 이런 export 방식은 다른 플랫폼으로 서비스를 이전할 때 고려해야하는 고민을 해결해 줄 수 있다.

{
  "db": [
    {
      "meta": {
        "exported_on": 1433545146635,
        "version": "003"
      },
      "data": {
        "posts": [
          {
            "id": 1,

            "markdown": "## Markdown 글 내용",
            "html":"<h2>글내용</h2>
 ... 생략...

호스팅

사이트를 운영하거나 블로그를 운영할 때 호스팅 가격 때문에 망설이는 사람들이 많다. 개인 서버를 운영할 수 있을 경우 개인 서버에 설치하면 되지만 대부분 개인 서버를 운영할 수 없기 때문에 돈을 지불하고 호스팅 서비스에 자신의 프로그램을 설치하여 운영한다. Jekyll로 만든 블로그는 GitHub Pages를 사용하여 무료로 호스팅을 할 수 있다. GitHub의 시스템이 Ruby 기반이고 Ruby 기반으로 만들어진 Jekyll을 사용하여서 운영할 수 있기 때문이다. 하지만 ghost의 경우는 Node.js를 기반으로 이것은 호스팅 서버가 있어야한다. 그럼 Node.js 호스팅을 하는 곳을 찾거나 Amazon Web Service와 같은 가상화 서비스에 설치하여 운영하면 되는데 시간과 돈이 꽤 필요하다.

나는 이런 문제를 해결하기 위해서 ghost를 buster로 GitHub Pages에서 운영하기로 결정했다. 외국 서비스들의 작명법은 늘 유머스럽다. 오래전에 유행하던 미국 드라마로 Ghost Busters가 있었는데 ghost의 서비스에 buster라는 이름의 서비스를 붙인것 같다.

buster는 ghost로 만들어진 컨텐츠를 GitHub Pages에 운영할 수 있는 static web service로 만들어주는 Python 라이브러리이다. Mac OS X에서는 기본적으로 Python이 설치되어 있기 때문에 다음 한 줄로 buster를 바로 설치할 수 있다.

pip install buster

다음은 ghost가 설치된 디렉토리 안에서 다음 명령어로 GitHub Pages에 운영할 수 있는 정적 웹 페이지를 만들어낼 수 있다..

buster generate

기본적으로 static/라는 디렉토리가 만들어지고 그 안에 정적 파일들이 만들어지게 되는데 이것을 git를 사용하여 GitHub 저장소에 push를 하면 GitHub Pages로 ghost를 운영할 수 있게 된다.
